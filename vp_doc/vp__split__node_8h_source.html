6 // split pipeline into multi branches. although all other non-Des nodes have the ability to split pipeline, but vp_split_node has more parameters and flexible behaviour.
7 // by default, other non-Des nodes split pipeline by simple copying, which just copy pointer of meta and push to next nodes, each next node handle the same meta allocated in heap(not thread-safe).
8 // in addition, other non-Des nodes push meta to next nodes without difference, each next node receive equal number of meta to previous node.
9 //
10 // in vp_split_node, we have below parameters to set:
11 // split_with_channel_index (false by default): if true, push meta according to its channel index, only those next nodes having the same channel index can receive meta.
12 // split_with_deep_copy (false by default) : if true, copy meta in heap and create new pointer, then push the new pointer to next nodes, next nodes have a totally different meta with previous.
13 //
14 // note: split_with_deep_copy = true will affect the performance of pipeline.
15 // above paramters can be set as true at the same time.
